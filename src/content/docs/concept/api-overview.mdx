## Validation object

A `Validation` object can be created several ways:
    - a [single](/concept/single-validation) `Validation` is created by the [constructor](/api/validation/constructor) function.
    - a [grouping](/concept/grouping-validations) `Validation` is created by the static methods [`Validation.group()`](/api/validation/static-methods/group) and [`Validation.glue()`](/api/validation/static-methods/glue)
    - also a [grouping](/concept/grouping-validations) `Validation` with nested (grouped), bound to form fields `Validation` objects is created by the [`Validation.profile()`](/api/validation/static-methods/profile) static method.
    - and lastly, a [single](/concept/single-validation) `Validation` is created by the [`Validation.clone()`](/api/validation/static-methods/clone) method from another `Validation`.

Nested (grouped) `Validation` objects are accessed through: 
    - the [`Validation().validations`](/api/validation/instance-properties/validations) property.
    - by form field names when created using the [`Validation.profile()`](/api/validation/static-methods/profile) method.

## Predicate object

A `Predicate` object can be created:
    - by the [constructor](/api/predicate/constructor) function.
    - cloned from another `Predicate` object when passed into the [constructor](/api/predicate/constructor#cloning-a-predicate) function.

`Predicate` objects added to validations can be accessed:
    - through the [`Validation().constraints`](/api/validation/instance-properties/constraints) property.
    - in [state callbacks](/concept/state-callbacks) through the [`ValidationResult`](/api/validation-result) object passed in.
