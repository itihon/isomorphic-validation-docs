A `Validation` object can be used as an event handler. When executed on the client side, it is just a wrapper over the [`Validation().validate()`](/api/validation/instance-methods/validate/) method and can be passed directly into the <a href="https://developer.mozilla.org/en-US/docs/Web/API/EventTarget/addEventListener" target="_blank">`addEventListener()`</a> method.

## Syntax

```js
Validation()()
Validation()(event)
element.addEventListener(type, Validation())
```

### Parameters

- `event` <sup>optional</sup>
<br/> An <a href="https://developer.mozilla.org/en-US/docs/Web/API/Event" target="_blank">`Event`</a> object. (Only the property <a href="https://developer.mozilla.org/en-US/docs/Web/API/Event/target" target="_blank">`target`</a> is used to validate predicate groups associated with the object it refers to.)
<br/> **Default**: `undefined`.

### Return value

See "Return value" of the [`Validation().validate()`](/api/validation/instance-methods/validate/#return-value) method.

### Exceptions

See "Exceptions" of the [`Validation().validate()`](/api/validation/instance-methods/validate/#exceptions) method.


## Description

Using a `Validation` object as an event handler is intended to reduce boilerplate.
